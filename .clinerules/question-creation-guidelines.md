## Brief overview
本规则文件规定了在创建测验题目时的类型、数量和质量要求，确保题目能够有效考查学习者的理解深度而非简单的事实记忆。

## 输出格式规范（重要！）

题目必须严格按照以下 JSON 格式输出，文件为 JSON 数组格式：

```json
[
  {
    "id": "ch02-001",
    "type": "single",
    "question": "题目内容",
    "options": [
      {"id": "A", "text": "选项1"},
      {"id": "B", "text": "选项2"},
      {"id": "C", "text": "选项3"},
      {"id": "D", "text": "选项4"}
    ],
    "answer": ["B"],
    "explanation": "详细解析"
  }
]
```

### 字段说明
- **id**: 字符串，格式为 "ch{章节号}-{题号}"，如 "ch02-001"
- **type**: 字符串，取值只能是 "single"（单选）、"multiple"（多选）、"boolean"（判断）
- **question**: 字符串，题目内容
- **options**: 数组，每个元素是对象，包含 id（A/B/C/D）和 text（选项文本）
- **answer**: 字符串数组，如 ["A"] 或 ["A", "B", "C"]
- **explanation**: 字符串，答案解析

### 题型特殊要求
- **单选题和多选题**：必须有4个选项（A、B、C、D）
- **判断题**：选项固定为 [{"id": "A", "text": "正确"}, {"id": "B", "text": "错误"}]
- **文件格式**：必须是 JSON 数组，不是对象，不包含 chapter/title 等额外字段

## 题目类型要求
- **避免纯事例细节题**：不应创建仅询问具体年份、数字、人名等纯事实性问题的题目
- **优先概念解析题**：重点考查核心理论、概念理解和原理应用
- **重视事例解读题**：将具体事例转化为对背后原理的理解和分析
- **强调理论阐述题**：深入考查概念间的关联性和系统性思维

## 题目数量规范
- **单章节题目总数**：通常为50道题，可根据章节重要性适当调整
- **概念题比例**：应占总题数的70-80%（约35-40题）
- **事例解读题比例**：应占总题数的20-30%（约10-15题）
- **纯事实题比例**：应控制在5%以下或完全避免

## 题目质量标准
- **多选题优先**：在适合的情况下优先使用多选题，考查综合理解能力
- **解释说明完整**：每个题目都应包含详细的解释，说明正确答案的理由
- **避免重复考查**：同一概念不应通过多个相似题目重复考查
- **注重应用层面**：题目应考查概念在实际场景中的应用，而非定义背诵

## 具体实施指导
- **事例题转化**：将"谁在什么时候做了什么"的问题转化为"这个事例说明了什么原理"
- **概念题深化**：不仅考查概念定义，更要考查概念的应用、影响和关联
- **雅各布·里斯名言处理**：必须包含名言来源、深层含义和象征意义的完整考查
- **复利效应和潜能蓄积期**：作为核心概念，需要从多个角度进行深入考查
